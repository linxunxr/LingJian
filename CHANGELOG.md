# 更新日志

本文件记录「灵鉴」各版本的变更内容。格式参考 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.1.0/)，按版本倒序排列。

发布时，CI 会自动从本文件提取对应版本段落，写入 `latest.json` 的 `notes` 字段，作为客户端更新提示中展示的更新日志。

---

## v0.1.16 - 2026-07-06

### 修复
- **更新内容显示为占位文本**：补全 v0.1.11 ~ v0.1.15 的更新日志段落，CI 提取 CHANGELOG 失败时不再回退到默认值「vX.X.X 更新内容」

## v0.1.15 - 2026-07-06

### 修复
- **关闭 Issue 弹窗不自动关闭**：输入版本号确认关闭后，Issue 已关闭但弹窗卡住不消失。改用 `try/finally` 保证清理，并防御 `labels` 为空时 spread 抛错

## v0.1.14 - 2026-07-06

### 修复
- **首页与关于页版本号滞后**：硬编码的 `v0.1.10` 改为动态读取 `getVersion()`，与更新器共用单一数据源，后续升级自动跟随

## v0.1.13 - 2026-07-05

### 新增
- **关闭 Issue 时记录解决版本号**：关闭弹窗输入挂机仙途版本号后，自动追加版本标签（如 `v0.9.19`）并添加「已在挂机仙途 vX.X.X 中标记为已处理」评论

### 修复
- **安装后仍显示旧版本号**：`tauri.conf.json` 与 `Cargo.toml` 版本号未同步，导致打包内部版本与 `package.json` 不一致

## v0.1.12 - 2026-07-05

### 新增
- **问题列表「已处理」Tab**：在「未处理」与「全部」之间新增按 `closed` 状态筛选的标签页

### 优化
- **COS 上传默认走全球加速域名**：加速域名为主、普通地域域名降为兜底，提升跨洲上传稳定性

## v0.1.11 - 2026-07-05

### 优化
- **SCF 响应解析失败诊断**：SCF 上游响应解析失败时输出原始响应体，便于定位问题

## v0.1.10 - 2026-07-04

### 新增
- **更新说明按 Markdown 渲染**：设置页「检查更新」看到的新版本日志不再显示 `###`、`-`、`**` 等源符号，而是渲染成正常的标题、列表与加粗（采用 marked 解析，适配深色主题）

### 优化
- **COS 上传加速触发阈值 3 → 1**：已购买全球加速流量包，普通域名失败一次即切换 `cos.accelerate.myqcloud.com`，避免在持续慢链路上做无效重试
- 同步刷新全站版本号显示（首页、设置页「关于」此前滞留在旧版本号）

## v0.1.9 - 2026-07-04

### 新增
- **分析详情页支持 Issue 操作**：从首页「最近分析」进入详情页后，可直接关闭/重开 Issue、添加评论、管理标签（此前这些操作只在首页问题列表中）
- 详情页头部显示 Issue 真实状态徽章（未处理/已关闭），关闭/重开按钮文案按远端真实状态切换
- 管理标签对话框回填 Issue 当前真实标签

### 优化
- 首页「最近分析」跳转带上 issue 号，详情页进入时即调用 `fetch_issue_info` 获取真实状态
- `IssueInfo` 结构补 `state`/`labels` 字段（带 serde default，旧版 SCF 不返回时降级为空）

### 修复
- 分析详情页同页换 id 不刷新：补 `route.query.id` 的 watch，并重置 Issue 操作本地状态

## v0.1.8 - 2026-07-04

### 修复
- **最近分析列表显示空记录**：Report 结构体漏加 `#[serde(rename_all = "camelCase")]`，导致 JSON 字段为 snake_case（report_id），前端按 camelCase（reportId）读取全部为 undefined

## v0.1.7 - 2026-07-04

### 优化
- 首页 keep-alive 缓存：从分析页返回首页不再重新加载，保留列表数据与滚动位置
- 最近分析加载增加诊断日志（F12 控制台可见 `[loadRecent]` 输出）

## v0.1.6 - 2026-07-04

### 修复
- 首页白屏：`list_recent_reports` 返回数据字段缺失时 `.slice()` 崩溃，加空值防御
- `list_issues` 返回异常结构时数组兜底，避免 `v-for` 遍历 undefined
- COS 上传网络参数调优，根治跨洲 write timeout
- COS 上传连续失败 3 次后自动切换全球加速域名（失败后自动回退）

### 新增
- 启用 DevTools（F12 调试控制台），便于排查运行时问题

## v0.1.5 - 2026-07-03

### 修复
- COS 关闭全球加速，改回普通地域域名（ap-guangzhou）
- COS 上传增强断点续传：分块重试 + 指数退避 + 碎片清理 + 进度可见
- 首页版本号显示修正

### 新增
- 启用 DevTools（F12 调试控制台），便于排查首页白屏等运行时问题

## v0.1.4 - 2026-07-03

### 修复
- 设置保存后首页仍提示配置不完整：apiKey 改存 settings.json 替代 keyring（部分 Windows 环境 keyring 写入静默失败）
- 应用图标显示为蓝色占位：用 tauri icon 重新生成全套图标，确保与源设计一致

### 优化
- COS 产物按版本分目录归档：安装包上传到 `/<版本>/` 子目录，`latest.json` 留根目录，避免所有版本混在一起
- COS 改用全球加速域名（CI 上传 + 客户端更新 + 用户下载均走加速接入点）

## v0.1.0 - 2026-06-09

首个可用版本：Path of Idle Immortals 日志分析工具。

### 新增
- 日志分析核心：通过 SCF 端点拉取日志、解析分类、本地 SQLite 落库与缓存
- 图表可视化：基于 Chart.js 的时间线图表与错误聚合面板
- 虚拟滚动列表，支持海量日志条目流畅浏览
- 问题列表：自动拉取 Issue、状态切换、点选分析
- Issue 操作：关闭/重开、评论、标签菜单（经 SCF 代理 GitHub）
- Markdown / JSON / CSV 多格式导出
- 凭证加密存储（系统钥匙串 keyring）
- 设置页连接验证与首次启动引导
- 键盘快捷键（Ctrl+F 搜索 / Esc 返回）
- 数据目录便携存储：跟随安装位置、可自定义、自动迁移
- 应用自动更新（Tauri Updater，腾讯云 COS 分发）

### 修复
- 下载日志后解析失败：兼容裸数组与包裹对象两种格式，补全 category 字段

### 优化
- 统一前后端类型契约
- 引导流程简化为两步（SCF URL + API Key）
- 移除 GitHub Token 凭证管理与直连逻辑，统一经 SCF 端点代理
